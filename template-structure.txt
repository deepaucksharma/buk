USAGE PATTERNS
==============

For Authors:
1. Start with Chapter Canvas (Section 6.1)
2. Fill invariant/uncertainty/evidence first
3. Type paths with G vectors
4. Define modes
5. Choose diagrams
6. Validate with linter

For Reviewers:
1. Check Canvas completion (Section 6.1, #15)
2. Validate G vector composition
3. Verify evidence properties
4. Ensure mode consistency
5. Run mental linter

For Readers (of finished chapters):
1. Identify invariants
2. Trace evidence lifecycle
3. Understand composition
4. Predict degradation
5. Apply to new domains

CHEAT SHEET
===========

Guarantee Vector: ⟨Scope, Order, Visibility, Recency, Idempotence, Auth⟩
Context Capsule: {invariant, evidence, boundary, mode, fallback}
Evidence Properties: {scope, lifetime, binding, transitivity, revocation}
Four Modes: Floor, Target, Degraded, Recovery
Five Diagrams: Guardian, Flow, Ladder, Compass, Knowledge/Data

Composition: meet(A, B) = weakest component wins
Upgrade: Need new evidence
Downgrade: Must be explicit and labeled
Capsule Ops: restrict, extend, rebind, renew, degrade

Invariants: Conservation, Uniqueness, Authenticity, Integrity, Order,
            Exclusivity, Monotonicity, Freshness, Visibility,
            Convergence, Idempotence, BoundedStaleness, Availability

Evidence Types: CommitCert, LeaseEpoch, ClosedTimestamp, ReadIndex,
                MerkleProof, VerkleProof, Signature, TEEAttestation
